# ISO 27001 Control Mapping (GRC Project)

## 📌 Overview

This project demonstrates practical implementation of ISO 27001 controls by mapping them to a real-world system (portfolio website).

## 🎯 Objective

To evaluate the current security posture and identify gaps based on ISO 27001 Annex A controls.

## 🧠 Methodology

* Selected key ISO 27001 controls
* Analyzed system against each control
* Identified gaps and weaknesses
* Suggested actionable improvements

## 📊 Controls Covered

* A.5.1.1 – Information Security Policy
* A.6.1.1 – Roles & Responsibilities
* A.8.1.1 – Asset Management
* A.9.2.1 – User Access Management
* A.9.4.1 – Access Control
* A.12.2.1 – Malware Protection
* A.12.4.1 – Logging & Monitoring
* A.13.1.1 – Network Security
* A.14.2.5 – Secure Development
* A.15.1.1 – Supplier Security

## 🔍 Key Findings

* Lack of access control mechanisms
* No logging and monitoring system
* Absence of formal security policies
* No asset inventory

## 🛠️ Recommendations

* Implement authentication system
* Introduce logging and monitoring
* Define security policies
* Use secure hosting and CDN

## 📌 Applicability

All selected controls are applicable to the portfolio system based on identified risks and system exposure.
